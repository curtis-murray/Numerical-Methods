4  Basic Algorithms

4.1 Motivation and Definition

When we want to solve a problem, especially one of a mathematical nature, we often need to follow a specific set of operations to achieve the desired result.

We call these sets of operations algorithms. In computer science and mathematics, we encode operations using our programming language so that the computer can do all the hard work for us!

The outcome of this lesson is not just to be able to understand a few algorithms but to understand the thought process behind how the algorithms you will come across were created.

4.2 Length

The first algorithm we will learn will allow us to find the length of a vector input.

Take the vector below:

Code 
x <- c(1,1,2,3,5,8)
x
[1] 1 1 2 3 5 8

We can count the number of elements to find the length of the vector. Here, we have six elements, so the length of x is six.

How did we work that out though?

Probably, you started counting the elements from left to right (or maybe right to left!). Starting with one, you progressively increment the running count by one each time you move to the next element.

When writing an algorithm, we must carefully consider the procedure that we need to follow to achieve the desired result. Usually, we can start with a basic example and then try to abstract our thought process to arrive at something general.

We might develop some pseudo code (human-readable code) as a starting point for the length algorithm.

Code 
initialise running total as 0.
start with first (left-most) element
increment running total by 1.
move to next element (to the right) and repeat
return running total

In the above pseudo-code, we have started to abstract and proceduralised a process that we might use to find the length of a vector.

Now let’s try and write that out in R.

Note

Because an algorithm will likely be used multiple times, we don’t want to write it out each time we need it! Instead, we can put the algorithm inside a function, so that whenever we need to apply an algorithm to an input, we can just call the function.

4.2.1 Basic Length Algorithm

Code 
basic_length <- function(x){
  # Initialise the running total as zero
  running_total <- 0
  
  # Loop over all elements of the input x
  for(element in x){
    
    # Increment the previous running total by 1
    running_total <- running_total + 1
  }
  
  # Return running total
  # Once all elements have been looped over
  return(running_total)
}

4.2.2 Testing

Now that we have created a function to find the length of an input vector let’s try it out!

Code 
basic_length(x)
[1] 6
Code 
basic_length(1:1000)
[1] 1000
Code 
basic_length(c("cat", "dog", "frog"))
[1] 3

4.2.3 A Note on R

R is not a particularly fast programming language. Running a for loop in R takes considerably longer than running an equivalent loop in Python and much longer than running in a language like C. However, R is still incredibly useful, so this should not immediately deter us from using R.

R was built with statisticians in mind, is open source, and has a fantastic package management system that allows R users to share code through the community easily.

There are many packages that can be imported, allowing you to use a function that someone else may have created (we will come across this in a future lesson). People might often use a faster language, like C, under the hood to speed up their packaged functions. As well as this, there are many base functions (functions that exist in the base environment, i.e., without any packages imported) that exist and are fast. Usually, they are either coded smartly or implemented in C. For example, the function length already exists in the base environment, so we never really needed to code it. We can pit our function basic_legnth against the base environment function length to see who finishes first!

Code 
system.time(basic_length(1:1e7))
   user  system elapsed 
   0.12    0.00    0.13
Code 
system.time(length(1:1e7))
   user  system elapsed 
      0       0       0

Uh-oh! Our function took around 0.1s, but the base environment took basically no time (rounded to 2 decimal places). Our function is orders of magnitude slower :(. I’m not exactly sure how the length function was implemented in R, but they likely did something smart, like check how big the memory block was for the object 1:1e7. This is like turning a book to its last page and reading the page number. So while our loop would have run the line running_total <- running_total + 1 10000000 times, their code probably just did one simple operation. This touches on a concept called computational complexity, but we can shelve that for another day and see a few more basic algorithms first.

4.3 Sum

In mathematical notation, if we have an \(n-\)dimesional vector \(\textbf{x} = \left(x_1, \cdots, x_n\right)\), the sum of the elements of \(\textbf{x}\) is

\[\begin{equation} \label{sum} \text{Sum}(\textbf{x}) = \sum_{i = 1}^n x_i . \end{equation}\]

As an exercise, write a function basic_sum that takes a vector input and returns the sum of the elements in the vector.

To start, think about the procedure that you might follow when calculating the sum of a vector, and come up with some pseudo code that generalises this procedure. Then try to implement that procedure in R.

4.3.1 Basic Sum

When you have tried, reveal a potential solution below by clicking the dropdown arrow. Remember, there is not just one way to find the sum!

Code 
basic_sum <- function(x){
  n <- length(x)
  sum <- 0
  for(i in 1:n){
    sum <- sum + x[i]
  }
  return(sum)
}
Code 
basic_sum(1:10)
[1] 55
Code 
basic_sum(c(-1,-4,2))
[1] -3

As we can see, the function basic_sum correctly finds the sum for the given input vectors above.

4.3.2 Explanation

When you take the sum of elements of a vector, say

Code 
x <- c(1,4,5,9)
x
[1] 1 4 5 9

You may progressively compute the cumulative sums: \(1+4 =5\), then \(5 +5 = 10\), and finally \(10 + 9 = 19\).

We may generalise this behaviour to

Code 
find the number of elements to sum over
start at the first element
find the cumulative sum
go to the next element, find the cumulative sum, and repeat
return the cumulative sum at the end

When we express the sum mathematically, as we did in equation\(\ref{sum}\), we can see that the summation takes place from \(i=1\) to \(i=n\), where \(n\) was the length of the vector \(\textbf{x}\). In the function basic_sum, we find \(n\) using the length function to define the indices we need to loop over.

We then successfully loop through the input vector and find the cumulative sum. Finally, we return the sum.

4.3.3 Alternative Basic Sum

Note that while we could loop over the indices 1:n of the input vector x and find the element at index i as x[i], we can just loop over the elements of the vector themselves.

Code 
alternative_basic_sum <- function(x){
  sum <- 0
  for(x_i in x){
    sum <- sum + x_i
  }
  return(sum)
}

Let’s compare how long these functions take to run and compare it to the base function sum.

Code 
system.time(basic_sum(1:1e7))
   user  system elapsed 
   0.20    0.00    0.21
Code 
system.time(alternative_basic_sum(1:1e7))
   user  system elapsed 
   0.13    0.00    0.12
Code 
system.time(sum(1:1e7))
   user  system elapsed 
      0       0       0

We can see that the first function, basic_sum, took the longest. The second function, alternative_basic_sum, was completed in less time. It is faster because fewer operations need to occur than the function basic_sum. Instead of finding the length of the vector, finding the indices, and indexing the vector by the indices, we simply loop over the elements. There are fewer steps for the computer to take, and hence, executes faster.

Of course, the base function sum completely demolishes our feeble attempt at writing the summation function. Fear not; our attempts are not futile. This is our first introduction to algorithms; we should not expect to challenge grand coding wizards without experience and emerge victorious.

4.4 Min

The next basic function that we will look at is the min function, which returns the minimum element of a vector. For example, the base R function performs as follows:

Code 
x <- c(3,pi,log(100))
min(x)
[1] 3

Try to find the minimum of a vector using an example. Then generalise and proceduralise the method that you used.

4.4.1 Basic Min Function

Reveal the code below once you have tried this for yourself:

Code 
basic_min <- function(x){
  min <- Inf
  for(x_in in x){
    if(x_i < min){
      min <- x_i
    }
  }
  return(min)
}

4.4.1.1 Explanation

In the function basic_min, we follow a similar procedure to the basic_sum algorithm, but instead of finding the cumulative sum, we find the running minimum, i.e., progressively find the minimum of the first few elements until we have looked through all elements. This is done by using an if statement, where we check to see if the current element is less than the minimum of the preceding elements. We initialise the minimum as Inf, (R’s representation of \infty). We could also initialise the minimum as x[1].

4.4.2 Bonus Exercise

What if we wanted the index of the minimum element, not the minimum element? For example, the index, or position of the minimum element in the vector \(\textbf{x} = \left(3, \pi, \log(100)\right)\) is \(1\), as the minimum element is \(3\), and \(x_1 = 3\). If you haven’t come across this before, we call this \[\underset{i}{\arg \min} (x_i),\] as this is the argument (here the index \(i\) is the argument), that minimises \(x_i\).

4.5.1 Mean

Find the mean of an input vector in a function basic_mean. Compare your function to the base R function mean.

Recall the mean \(\bar{\mathbf{x}}\) of \(n-\)dimensional vector \(\mathbf{x}\) is defined as

\[ \bar{\mathbf{x}}= \dfrac{\sum_{i=1}^n x_i}{n}. \]

4.5.2 Solution

Unfold the solution when you are ready.

Code 
basic_mean <- function(x){
  mean <- basic_sum(x)/length(x)
  return(mean)
}

4.5.3 Standard Deviation

The standard deviation \(\sigma\) of a sample is defined as

\[ \sigma(\mathbf{x}) = \sqrt{\frac{\sum_{i=1}^n(\bar{x} -x_i)^2}{n-1}}, \] where \(\mathbf{x}\) is an \(n-\)dimensional vector with mean \(\bar{\mathbf{x}}\).

Write a function basic_sd to find the standard deviation of an input vector. You may use your function basic_mean to help. Compare how long your solution takes to run compared to the base function sd.

4.5.3.1 Solution

Unfold the solution when you are ready.

Code 
basic_sd <- function(x){
  n <- length(x)
  mean <- basic_mean(x)
  # Sum of the squared differences (errors)
  sse <- 0
  for(x_i in x){
    sse <- sse + (mean-x_i)^2
  }
  sd <- sqrt(sse/(n-1))
  return(sd)
}

4.5.4 Mode (Harder)

The mode of a vector is the value that appears most frequently). For example, the mode of the vector \[ x = (1,2,2,4)\] is \(2\) since this occurs the most frequently.

Write a function basic_mode that finds the mode of a vector.

Assume that the vector is unimodal, i.e. there is only one mode. The vector \[ x = (1,1,2,2) \] is not unimodal, since there is no single most frequently occurring value. We call this type of vector bimodal.

4.5.4.1 Solution

A very basic (and inefficient implementation of basic_mode is given below)

Code 
basic_mode <- function(x){
  
  # Function to find how many times 
  # an element x_i appears in the vector x
  times_appears <- function(x_i,x){
    count <- 0
    for(x_i_prime in x){
      if(x_i_prime == x_i){
        count <- count + 1
      }
    }
    return(count)
  }
  
  # Set up a variable to find how many times the 
  # mode appears (initialised at zero)
  max_count <- 0
  # Progressively find the mode as 
  # the variable that appears more
  # than max_count 
  for(x_i in x){
    x_i_count <- times_appears(x_i,x)
    if(x_i_count > max_count){
      max_count <- x_i_count
      mode <- x_i
    }
  }
  return(mode)
}
